### Alarm & Snooze — Analysis Done
- James flagged alarms and snooze not working properly
- Root cause: web app limitation — `setInterval` only runs when tab is open and active
- Alarms fail when: tab closed, screen off, app backgrounded, phone locked
- Specific bugs: AudioContext created at module load (blocked on iOS), no Browser Notification API, snooze expiry silent (doesn't re-alarm), refire bug on un-mark
- **Real fix: Capacitor + `@capacitor/local-notifications`** — native notifications work when app is fully closed
- Interim fixes possible: Browser Notification API, lazy AudioContext, snooze re-trigger
- Analysis saved: `/workspace/memory/alarm-fix-plan.md`
